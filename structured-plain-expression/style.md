# Language: Expression, Voice, Stance · 语言风格：表达、语气、姿态

The rules of execution for structured plain expression. Most carry an example
(✓) and a counter-example (✗).
本文件给出"结构化白话"的落笔规则。多数规则配范例（✓）与反例（✗）。

---

## Expression | 表达

1. **A plain recap after every table.** A table never hangs alone; one line
   lands its point — "in one line / read it as / the key".
   **每张表后必有白话收束**。表不孤悬；一句话落点——"一句话／读法／关键"。
   - ✓ after a comparison table: *"In one line: A for text, B for scale."*
     在对比表后：*"一句话：文本选 A，规模选 B。"*
   - ✗ a table followed immediately by the next heading.
     表格后直接进下一个标题。

2. **Intuition before terminology.** Give a concrete picture first, the formal
   definition second; let the reader build a mental model in three seconds.
   **直觉先于术语**。先给画面，再给定义；让读者三秒建立心智模型。
   - ✗ *"Attention stickiness: the temporal stability of the sustained-attention
     ratio."*
     "关注粘性：持续关注比例的时序稳定性。"
   - ✓ *"Attention stickiness ≈ a fan's retention — high = regulars come daily,
     low = passers-by glance once. (Formally: ...)"*
     "关注粘性≈粉丝留存——高＝老粉天天来，低＝路人看一眼就走。（学术定义：……）"

3. **Concrete over vague.** Give a number or a shape, not "significantly" or
   "greatly improved".
   **具体胜于空泛**。给数字或形态，而非"显著""大幅提升"。
   - ✗ "performance improved markedly" — ✓ "Alpha > 1%/month, t > 2".
     ✗ "性能大幅提升" — ✓ "Alpha > 1%/月，t > 2"。

---

## Voice & rhythm | 语气与节奏

4. **Dialogic self-Q&A.** Use the second person; at each turn, ask the reader's
   own question, then answer it. Avoid the "this paper studies…" register.
   **对话式自问自答**。用第二人称；在转折处替读者问出他的问题，再回答；不用
   "本文研究了……"的腔调。
   - ✓ *"It works — but why? And if there is an order, who moves first?"*
     "它有效——但为什么？若有先后，谁先动？"
   - ✗ *"This section first argues existence, then discusses the mechanism."*
     "本节首先论证存在性，继而讨论机制。"

5. **Functional emoji as navigation, not decoration.** Each marker carries one
   fixed meaning across the whole document (e.g. 📌 anchor · 📊 figure · ⚠️
   caution · ⭐ key); never mood emoji (🎉💯🚀).
   **功能性 emoji 作导航，非装饰**。每个标记全文固定一义（如 📌 定位 · 📊 图表 ·
   ⚠️ 注意 · ⭐ 重点）；不用情绪 emoji（🎉💯🚀）。

6. **Status symbols, subject declared.** For progress use fixed marks (✅ done ·
   🟡 partial · ⚪ skipped · 🔴 blocked) and state their subject up front, so they
   are never ambiguous.
   **状态符号，先声明主语**。进度用固定符号（✅ 完成 · 🟡 部分 · ⚪ 跳过 · 🔴
   阻塞），并在开头声明主语，避免歧义。

## Stance | 姿态

7. **Say it as you would to a colleague.** Plain and direct; cut jargon walls;
   readable but not cute — no memes, no padded enthusiasm.
   **像对同事那样讲**。平实、直接；拆掉术语墙；好读但不卖萌——无梗、无注水的热情。

8. **End each unit on one line.** Every section or unit closes with a single
   takeaway the reader can carry away.
   **每个单元以一句话收束**。每节／每单元都以一句可被带走的要点结尾。

## One-pass clean | 一键清洗流程（润色旧文时）

1. Hoist a 30-second overview to the first screen. | 把"30 秒概览"提到第一屏。
2. Turn every parallel / contrast / mapping prose block into a table. |
   把每段并列／对比／映射的散文改成表格。
3. Add a one-line recap under every table. | 给每张表补一句白话收束。
4. Put an intuition (an analogy) before each piece of terminology. |
   在每个术语前补一个直觉（比喻）。
5. Replace "see above / as mentioned" with specific section or file numbers. |
   把"见上文／如前所述"换成具体小节或文件号。
