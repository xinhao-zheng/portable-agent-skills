# Structured Plain Expression · 结构化白话研究帖

## What this is | 这是什么

In one line: **a way to write a multi-figure, data-backed study as a long post the
reader grasps in 30 seconds, follows figure by figure without ever getting lost, and
finishes able to retell — opening on the finding, advancing as a chain, pinning
every number to its source, stacking insight on insight, and closing on one line
worth carrying away.**
一句话：**一种把"有多张图表与数据支撑的研究"写成长帖的方法——读者 30 秒抓住结论，
逐图往下不迷路，读完能复述：开篇即结论，证据成链，每个数字钉住出处，洞见层层垒高，
收在一句可带走的话上。**

Its register is **austere in logic, plain in language**: structure carries the
argument, plain words carry the reader. It takes the austere-analytical-prose
discipline (one narrow claim → causal advance → an honest limit → an aphoristic
coda) but tells it in sentences a colleague would say — explaining each term, not
hiding behind it. It is neither a report (thorough but unread) nor a ramble
(readable but formless).
它的调性是**逻辑冷峻、语言白话**：用结构承载论证，用白话承载读者。它取冷峻思辨学术体的
纪律（一条收窄的主张 → 因果推进 → 诚实的局限 → 格言收束），却用同事会说的话讲出来——
把每个术语讲开，而非躲在它后面。它既不是报告（详尽但没人读），也不是随笔（好读但不成体系）。

Its opposite is two failures. One is the **report dump**: every chart pasted in,
every number listed, terms untranslated, the point buried — looks rigorous, closed
in five seconds. The other is the **loose ramble**: readable but with no spine,
no sourced number, no falsifiable claim — nothing a specialist can lean on.
它的反面是两种失败。其一是**报告堆砌**：贴满图、罗列数、术语不翻、要点埋底——看着严谨、
五秒被关掉。其二是**松散随笔**：好读却没有脊柱、没有带出处的数字、没有可证伪的主张——
内行无可凭借。

## Style fingerprint | 风格指纹（八项，与 logic.md 八步一一对应）

1. **Title is the finding** — the title states one falsifiable conclusion, not a
   topic label, with a one-line front matter (what this studies; a 非投资建议
   note when the subject is a market).
   **标题即结论**——标题陈述一条可证伪的结论、而非题材标签，紧跟一行 front matter
   （在研究什么；涉市场则加*非投资建议*）。
2. **Overview first** — within the first screen, one "30-second lookup" table
   (reader's question / our answer / which figure), closed by a one-line thesis;
   stop there and still leave with 80%.
   **概览先行**——第一屏一张"30 秒速查"表（读者会问／我们的答案／看哪张图），一句话
   主张收束；只读第一屏也带走八成。
3. **Problem before method** — open on the anomaly you found (with a small
   before/after table), then why this metric sees what ordinary tools can't, then a
   route table of how you'll check it.
   **先问题，后方法**——先摆发现的反常（配前后对照小表），再讲这个指标比常规手段多看到
   什么，再给一张排查路线表。
4. **Evidence as a chain, four beats per figure** — figure by figure: how to read it
   → what it shows → how it links to the next → one bolded takeaway; adjacent
   figures must not be swappable.
   **证据成链，每图四拍**——逐图：怎么看 → 看出什么 → 和下一张怎么接 → 一句加粗收束；
   相邻两图不可互换。
5. **Quantified replay + an honest gap** — compress the core pattern into one data
   table (extremes / percentile / aftermath), recap under it, surface the caliber
   trap or lead-lag yourself, then corroborate with real-world cases — each number
   sourced.
   **量化复盘 + 诚实 gap**——核心规律压成一张数据表（极值／分位／后续），表后收束，主动
   标出口径陷阱或领先滞后，再用真实世界案例佐证——每个数字带出处。
6. **Insight stacked in layers** — baseline → mechanism / why → limit (can /
   cannot); each layer is one insight, run as four beats, closed on a bolded line,
   its objection anticipated.
   **洞见层层垒高**——基线 → 机制／为什么 → 边界（能／不能）；每层一个洞见、四拍展开、
   一句加粗收束、并预判反对。
7. **A causal chain for every surprise** — give A → B → C for anything
   counter-intuitive, answer "why X not Y" head-on; lead plain, unpack each term,
   an analogy where it helps (sentence-level habits live in style.md).
   **每个意外都给因果链**——反直觉处给 A → B → C，正面回答"为什么是 X 不是 Y"；白话
   先行、术语拆开、必要处一个比喻（句段级习惯见 style.md）。
8. **Aphoristic close, ringing back** — each section ends on one bolded carried-away
   line; the whole closes on one sharp contrastive aphorism echoing the opening (or
   the prior piece). No emoji, no clickbait; 非投资建议 when the subject is a market.
   **格言收尾、首尾呼应**——每节一句加粗、可带走的话；全文以一句锋利的对照式格言收尾，
   呼应开篇（或前作）。无 emoji、无标题党；涉市场则标*非投资建议*。

## How to use | 怎么用

**Writing new** — read [logic.md](logic.md) for the advance order (title → lookup
→ problem & method → evidence chain → quantified replay → stacked insight →
coda), then write per [style.md](style.md): skill docs in the austere voice
(Voice A); the post itself in the structured plain voice (Voice B). Mark where each
figure goes; reference figures as `FigN` in the body, but keep section headings
free of figure codes.
**新写稿**——先读 [logic.md](logic.md) 定推进顺序（标题 → 速查 → 问题与方法 → 证据链 →
量化复盘 → 层层洞见 → 收束），再依 [style.md](style.md) 落笔：skill 文档用冷峻体
（Voice A）；帖子正文用结构化白话体（Voice B）。标清每张图插在哪；正文用 `FigN` 引用图，
但小节标题不带图号。

**Polishing a draft** — first apply the language habits in [style.md](style.md)
(plain-first, recap under every table, term unpacked on first use, bolded
takeaways, sourced numbers), then check the skeleton per [logic.md](logic.md)
(chain not pile, four beats per figure, objection answered, coda that rings back).
**润色草稿**——先按 [style.md](style.md) 套语言习惯（白话先行、表后收束、术语首次拆开、
加粗收束、数字带出处），再按 [logic.md](logic.md) 检查骨架（成链非堆、每图四拍、
预判反驳、收尾呼应）。

| File | Role |
|------|------|
| [logic.md](logic.md) | Advance order, section skeleton, self-check · 推进顺序、章节骨架、自检 |
| [style.md](style.md) | Skill-doc voice + post-output voice · skill 文档体 + 帖子文体 |

## One contrast | 一个最小对照

> Before (report dump) — *"F=60,H=20 OOS mean IC=0.66，历史分位 100%，峰后 40 日 IC
> −0.53，前 10% 回撤 −15.1%……"*（把一张表念出来，数字无机制、无出处、无收束）

> Before (loose ramble) — *"中长期 IC 最近特别高，看着像要变盘，应该是抱团了吧。"*
> （好读，但无脊柱、无数字、无可证伪的主张）

> After (this skill) — *"一个平时没用的指标，突然变准了——`F=60,H=20` 九年均值才
> −0.027（时正时负、平均还略亏），最近却冲到 0.66。这不是因子复活：要让"两三个月前谁强"
> 重新可预测，只有一种情形——资金把相对格局焊死，也就是抱团。历史四次这样冲顶，随后一两个
> 月 IC 都翻到 −0.5 上下（本机测算）。一句话：它量的是拥挤，不是余温。"*
> （白话先行 + 机制因果 + 带出处的数字 + 一句加粗收束）

The difference is entirely: the number-wall and the formless guess both yield to a
plain-first chain — claim, mechanism, sourced number, takeaway — that a layperson
reads and a specialist trusts.
差别全在：数字墙与无形的猜测，都让位于一条白话先行的链——主张、机制、带出处的数字、收束——
外行读得懂，内行信得过。
