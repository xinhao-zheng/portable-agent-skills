# Structured Plain Expression · 结构化白话表达

## What this is | 这是什么

In one line: **a way of writing long documents that is at once professional and
readable — the whole grasped in one screen, any detail found in two minutes, and
the reader able to act the moment they finish.**
一句话：**一种把长文档写得既专业又可读的方式——一屏看到全局，两分钟找到细节，
读完即可动手。**

It refuses two opposites: the bloated eight-part essay (professional but unread)
and the loose ramble (readable but formless). Structure carries the logic; plain
language carries the reader.
它拒绝两个反面：八股（专业但没人读）与松散（好读但不成体系）。用结构承载逻辑，
用白话承载读者。

## Style fingerprint | 风格指纹（七项，与 logic.md 七步一一对应）

1. **Overview first** — the first screen is a map (diagram, big table, or
   one-minute lookup); 80% is graspable in 30 seconds.
   **概览先行**——第一屏是一张地图（图、大表或一分钟速查），30 秒看懂八成。
2. **Three-layer disclosure** — overview → detailed body → appendix; the reader
   goes only as deep as the need.
   **三层渐进披露**——概览 → 详细主体 → 附录；读者按需下探。
3. **Form follows relation** — a table for parallel / contrast / classification
   / mapping; a numbered list for sequence; short prose for a single thread.
   **形式随关系**——并列／对比／分类／映射用表格；时序用编号列表；单一叙事用短段。
4. **Four-part unit** — every task or step runs: why → how → how to judge →
   expected.
   **四件套单元**——每个任务／步骤走：为什么 → 怎么做 → 怎么判断 → 预期。
5. **Anticipate objections** — after a claim, raise the rebuttal yourself and
   answer it.
   **预判反对**——抛出结论后，主动提出反驳并回答。
6. **Explicit cross-reference** — point to a specific section or file, never
   "see above".
   **明确交叉引用**——指向具体小节或文件，不用"见上文"。
7. **Close with a self-check** — end on a short checklist.
   **以自检收尾**——结尾放一张简短清单。

## How to use | 怎么用

**Writing new** — first read [logic.md](logic.md) for the build order (overview

## One contrast | 一个最小对照

Before (prose wall) · 原句（散文堆砌）:

> There are several ways to extract text: tool A is light but weak on scans;
> tool B is full-featured but heavy to install; tool C is simple but has an old
> interface; for scans you need an OCR step.
> 提取文本有几种方式：A 工具轻量但对扫描件弱；B 工具功能全但安装重；C 工具简单
> 但接口老；扫描件还得走 OCR。

After (table + recap) · 改后（表格 + 收束）:

| Tool | Best for · 适合 | Trade-off · 代价 |
|------|----------------|------------------|
| A | plain text · 纯文本 | weak on scans · 扫描件弱 |
| B | full processing · 全功能 | heavy install · 安装重 |
| C | light edits · 轻量 | old interface · 接口老 |

**In one line: text → A; scans → OCR; otherwise as needed.**
**一句话：纯文本选 A；扫描件走 OCR；其余按需。**

The difference is entirely: a parallel relation moved from prose into a table,
closed by a one-line recap — the reader sees the whole choice at a glance.
差别全在：把并列关系从散文搬进表格，再用一句话收束——读者一眼看清全部选择。
