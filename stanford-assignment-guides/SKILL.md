# Stanford Assignment Guides · Stanford 作业指南

## What this is | 这是什么

In one line: **one closed assignment model projected into three views — what the
source says, what knowledge it tests, and what the student must deliver — without
letting any view invent an answer.**
一句话：**把一个闭合的作业事实模型投影成三种视图——原题说什么、考什么知识、学生须
交什么；任何视图都不得越界发明答案。**

Its opposite is three independently written summaries. Their titles may agree
while points drift, subproblems disappear, and a “helpful hint” becomes a
solution. One source model prevents three versions of the truth.
它的反面是三份各写各的摘要：标题相同，分值漂移，小题消失，“友好提示”变成答案。
一个来源模型，阻止三种真相。

## Style fingerprint | 风格指纹（八项，与 logic.md 八步一一对应）

1. **Canonical assignment identity** — resolve course, assignment, live source
   set, and output convention before reading deeply.
   **规范作业身份**——深读前解析课程、作业、真实来源集与输出约定。
2. **Closed source graph** — follow actual document dependencies and bind the
   rendered assignment, scoring metadata, submission rules, and code interfaces.
   **闭合来源图**——沿真实文档依赖，绑定渲染题面、计分元数据、提交规则与代码接口。
3. **One truth model** — order, subproblem identity, type, points, requirements,
   examples, and submission rules are reconstructed once.
   **单一事实模型**——题序、小题身份、类型、分值、要求、示例与提交规则只重建一次。
4. **Faithful bilingual projection** — every operative source block appears in
   its source language unchanged; when that language is not Chinese, a semantic-
   block Chinese translation follows.
   **忠实双语投影**——每个有效来源块保留源语言原文；源语言非中文时，随后给语义块级中文翻译。
5. **Knowledge projection** — derive the overview from the closed assignment:
   capability, concepts, question mapping, course relation.
   **知识投影**——从闭合作业推出 Overview：能力、概念、题目映射、课程关系。
6. **Action projection** — orient each parent problem, then give every smallest
   graded unit its source ID, score/type, prompt meaning, required action,
   deliverable, constraints, and one non-solution warning.
   **行动投影**——先定位每道大题，再为每个最小计分单元给来源 ID、分值/类型、题意、所需
   行动、交付物、限制与一个无答案提醒。
7. **Serial artifact chain** — bilingual source view → overview → action guide;
   generate requested views without reversing dependencies.
   **串行工件链**——双语来源视图 → Overview → 行动 Guide；生成所请求视图但不逆转依赖。
8. **Cross-artifact reconciliation** — identity, scoring, dates, terminology,
   coverage, naming, and the no-generated-solution boundary pass across all outputs.
   **跨工件对账**——身份、计分、日期、术语、覆盖、命名与“不新增答案”边界须在所有产物间通过。

## How to use | 怎么用

**Building one artifact** — read [logic.md](logic.md), discover and close its
upstream sources, then render only that view per [style.md](style.md).
**生成单件**——先读 [logic.md](logic.md)，发现并闭合其上游来源，再按 [style.md](style.md)
只生成该视图。

**Building the bundle** — create and audit the bilingual source view, derive and
audit the Overview, then derive the Chinese action guide. Work serially; if the
user already requested the bundle, do not insert confirmation pauses.
**生成整套**——先生成并审计双语来源视图，再推出并审计 Overview，最后推出中文行动 Guide。
串行执行；用户已明确要整套时，不强插确认停顿。

| File | Role |

| [style.md](style.md) | Skill-doc voice + three output registers · Skill 文档体 + 三种输出声线 |

## One contrast | 一个最小对照

> Before (three summaries) — *Problem 3 is about search; implement A\*; 10 points.*
> 原做法（三份摘要）：Problem 3 关于搜索；实现 A\*；10 分。

> After (this skill) — *The authoritative prompt fixes the wording, the scoring
> source fixes graded units, the rendered document fixes visible order, and the
> three artifacts expose respectively the full text, knowledge map, and required
> deliverable — all with the same total.*
> 改后（本 skill）：权威题面钉住措辞，计分来源钉住计分单元，渲染文档钉住可见顺序；
> 三份工件分别呈现完整文本、知识地图与交付要求——总分始终一致。

The difference is entirely: three documents cease to be three retellings and
become three projections of one audited assignment.
差别全在：三份文档不再是三次转述，而是一个经审计作业的三次投影。
