# Stanford Assignment & Unit Visuals · Stanford 作业与单元配图

## What this is | 这是什么

In one line: **a visual argument in which every image shows one object changing
for one reason into one result — the overview carries the learning chain; each
section visual makes one mechanism visible.**
一句话：**一种视觉论证：每张图只让一个对象因一个原因变成一个结果——总览图承载学习链，
章节图把一个机制变得可见。**

Its opposite is the polished information wall: headings in boxes, arrows between
them, formulae as decoration, no object undergoing change. It looks organized.
It explains nothing.
它的反面是精致的信息墙：标题装进框，箭头连起来，公式作装饰，却没有对象发生变化。
它看起来有组织。它什么也没解释。

## Style fingerprint | 风格指纹（八项，与 logic.md 八步一一对应）

1. **Scope fixed first** — assignment or unit; overview, section set, or both;
   requested scope fixes the asset ledger before generation.
   **先定范围**——作业或单元；总览、分章或两者；请求范围在生图前固定资产台账。
2. **Closed content map** — validated analysis plus the matching overview/summary,
   or equivalent source artifacts, control boundaries, learning chain, and terminology.
   **内容图闭合**——已验证 Analysis 加匹配的 Overview/Summary，或等价来源工件，控制边界、
   学习链与术语。
3. **One main object** — each image receives one identifiable device, scene,
   object, or spatial metaphor; no object, no generation.
   **一个主对象**——每图只分配一个可指认装置、场景、对象或空间隐喻；无对象，不生成。
4. **Causal storyboard** — who changes, why, and what it becomes; arrows carry
   verbs and serve the object rather than replace it.
   **因果 storyboard**——谁在变、为何变、变成什么；箭头带动作词，服务对象而非替代对象。
5. **Inherited or declared visual system** — identity, aspect ratio, palette,
   hierarchy, typography, and naming are discovered once and held across the set.
   **继承或声明视觉系统**——身份、画幅、配色、层级、字体与命名发现一次并贯穿整组。
6. **Instructional generation** — default to AI image generation for raster
   illustration; reject formula, flowchart, and marketing-poster walls.
   **教学式生成**——位图插图默认 AI 生图；拒绝公式墙、流程图墙与营销海报墙。
7. **Rendered inspection** — inspect actual assets, not prompts: text, overlap,
   crop, sequence, identity, object, and causal legibility.
   **渲染后检查**——检查实际工件，不检查提示词自我感觉：文字、遮挡、裁切、顺序、身份、
   对象与因果可读性。
8. **Repair before release** — repair the smallest upstream cause, save to a
   deterministic ledger, and reconcile count, naming, and coverage.
   **发布前修复**——修最小上游原因，按确定台账保存，并对账数量、命名与覆盖。

## How to use | 怎么用

**Creating fresh** — read [logic.md](logic.md) to close the content map and define
the visual system and asset ledger, then generate and inspect per
[style.md](style.md). Use the available image-generation capability for bitmap
assets; do not substitute a diagram merely because it is easier to lay out.
**新建配图**——先读 [logic.md](logic.md) 闭合内容图、定义视觉系统与资产台账，再按
[style.md](style.md) 生成与检查。位图资产使用可用图像生成能力；不得因流程图更易排版就偷换产物。

**Repairing a set** — inspect every existing asset, classify the failure as
content, metaphor, composition, text, identity, or consistency, then repair the
smallest upstream cause. Do not patch labels onto a conceptually wrong image.
**修复整套**——检查每张现有工件，把失败归为内容、隐喻、构图、文字、身份或一致性，再修
最小上游原因。概念错的图，不靠叠字挽救。

## One contrast | 一个最小对照

> Before (flowchart wall) — *Evidence → posterior → prediction*, three boxes and
> two arrows.
> 原做法（流程图墙）：Evidence → posterior → prediction，三个框、两根箭头。

> After (this skill) — *A translucent evidence filter narrows a field of possible
> worlds; a posterior spotlight brightens the survivors; one plain caption says:
> “证据不是给答案——它只是让不可能者退场。”*
> 改后（本 skill）：一张透明证据滤镜收窄 possible worlds；posterior 聚光灯照亮幸存者；
> 一句白话说明：“证据不是给答案——它只是让不可能者退场。”

The difference is entirely: boxes name stages; a visual metaphor makes the
mechanism visible.
差别全在：方框给阶段命名；视觉隐喻让机制可见。
